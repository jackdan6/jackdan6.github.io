ppt讲解技巧

1、已经完成的事一栏量一定要充足

2、在有足够的量的前提下，要准备三个重点讲的，这三个重点一定要精心准备，且一定是已经体现了用户价值，不能讲他以后的价值。概括来说，量足够，且有突出重点

3、在说有价值的事时，一定要让对方觉得很难，经过了很大的努力（需要准备具体），最终解决问题，然后这个问题带来了巨大的用户价值（或者说经过了系统测试）。一定体现出用户价值大

4、成果物一定要多，没人看但是给人直观感觉很好

5、管理数据统计：

好的方面，一定要说明之前怎么样不好，经过那些努力，最终结果是怎样的，解决了那些问题，以及产生的价值

坏的方面，一笔带过，千万不要陷阱去解释，具体可以讲到某个点，今后的打算，如何通过努力攻克这个问题，需要注意的是，讲坏的方面，一定要给出解决思路

6、管理统计数据里面，标红的指标，一定要想好今后解决的办法。遗留到现场的问题，一定要给出今后解决方法。

7、讲坏的方面时的几个大忌：老板决策，环境不够，挑战规则

8、下季度的计划：需要着重解决的问题，要攻克的难点问题

9、阻碍：不要提开发模式的问题，不要挑战规则。可以说一些小问题，比如笔记本配置问题。或者说一些关键环节的资源支撑问题

10、价值最大和投入精力最多的事应该是同一件事

11、一定要让对方感觉到团队有很多产出，细致到每个组员产出那些东西。领导感受组织能力，就是看你是否能把事情拆解的足够细致，是否让下属工作饱满，给下属安排的工作一定是多维度的，一定不能单一维度。体现出组内安排的很到位

12、引导别人的提问思路，向自己强项转移

13、准备的三个突出事情一定要详细准备，精确的每一步

14、说话要简洁，有效

需要准备的三个事：

三个重点or，研发指标准备，下季度要攻克的事

一种ceph下的数据迁移方法

1、重新创建一套故障域、并给指定root绑定一个ruleset

2、使用命令：ceph osd pool set $pool_name crush_rule $rule_name给资源池重新指定规则，即可完成数据迁移

备注：这次测试仅仅实验了副本资源池，ec资源池还未尝试

2、hexo相关命令备注：

注意：hexo命令现在只能在hexo顶层目录下执行

hexo new ‘filename’ 创建一个新的hexo文件，然后直接进去编辑就可以了

hexo g(generate) 根据配置文件更新本地blog

hexo d(deploy) 上传博客到github上

hexo clean 清除本地配置

hexo s(serve) 运行本地blog，然后就可以登陆https://jackdan6.github.io来访问本地blog
